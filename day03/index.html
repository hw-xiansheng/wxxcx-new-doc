    # 一、uniApp介绍

    # 1、uniApp介绍

    问题：

    1、uniApp是什么

    2、什么是混合开发技术

    总结：

    1、uniApp是什么

    uni-app 是一个使用 Vue.js 开发所有前端应用的框架。

    开发者编写一套代码，可发布到 iOS、Android、H5、以及各种小程序（微信/支付宝/百度/头条/QQ/钉钉/淘宝）、快应用等多个平台。

    文档地址：https://uniapp.dcloud.net.cn/

    2、什么是混合开发技术

    混合开发技术：对应的是一套代码可以最终通过打包生成不同平台的代码，发布到不同平台，uniApp就是这种。

    类似uniApp的这种混合开发技术还有：

    • 京东前端团队开发 的Taro，文档点击 (opens new window)
    • Flutter技术栈： 文档点击 (opens new window)
    • ReactNative技术栈： 文档点击 (opens new window)

    注意： Taro类似uniApp最终可以打包成h5、各个平台的小程序、App(安卓,ios) ；但是 Flutter和ReactNative只能打包最终生成App（安卓,ios）

    问题：

    1、如何安装HbuilderX

    2、如何安装scss/sass编译

    1、如何安装HbuilderX

    • 下载HbuilderX

      • 访问 HBuilderX 的官网首页 https://www.dcloud.io/hbuilderx.html

      • 点击首页的 DOWNLOAD 按钮

    • 安装HbuilderX

      • 将下载的 zip包 进行解压缩; mac版本直接双击dmg文件
      • 将解压之后的文件夹，存放到纯英文的目录中（且不能包含括号等特殊字符）
      • 双击 HBuilderX.exe 即可启动 HBuilderX

    • 找到插件地址：https://ext.dcloud.net.cn/plugin?id=2046

    • 点击使用 HBuilderX 导入插件

    问题：

    1、HbuilderX方式创建的目录含义如何

    总结：

    1、HbuilderX方式创建的目录含义如何

    uni_project
    ├─.hbuilderx   		# 编辑器配置
    ├─pages				# 页面文件夹
    │  └─index			# 首页文件夹，一个页面一个文件夹，底下一个同名的.vue文件
    ├─static  		    # 静态文件夹
    └─unpackage			# 打包编译产物(dev和build的都在下面)  注意这个目录一般git需要忽略掉

    ├─App.vue 			# 根组件App.vue  应用的一些生命周期写在这里
	├─index.html		# 根html模板
	├─main.js 			# 入口文件
	├─manifest.json		# 项目配置(h5,app,各个小程序的配置都在这里)
	├─pages.json 		# 配置文件(全局配置和局部配置都在这里)
	├─uni.scss			# uniApp内置的scss样式，所需要安装编译软件编译

    问题：

    1、模板语法注意些什么？

    2、顶层对象是谁？

    总结：

    1、模板语法注意些什么？

    • 文档位置：https://uniapp.dcloud.net.cn/tutorial/page.html

    • template里面的组件标签类似微信小程序里面的组件标签一样，同样不可以写html

    • script里面如果是页面写小程序的页面生命周期onLoad、onShow、onUnload；如果是组件写vue的生命周期

    • style里面可以使用lang="scss"来实现样式预处理、使用scoped实现隔离、使用rpx单位实现响应式

    2、顶层对象是谁？

    • 顶层对象是uni，可以通过在main.js里面通过对uni.xx= xxx实现顶层挂载，在组件里面通过uni.xx进行读取调用

    # 3、条件编译

    • **以 #ifdef 或 #ifndef 加 %PLATFORM% 开头，以 #endif 结尾

      • #ifdef：if defined 仅在某平台存在
      • #ifndef：if not defined 除了某平台均存在
      • %PLATFORM%：平台名称
      • 条件编译是利用注释实现的，在不同语法里注释写法不一样，js使用 // 注释、css 使用 /* 注释 */、vue/nvue 模板里使用 <!-- 注释 -->

    • 实例如下

      <!--  #ifdef  H5 || APP-PLUS -->
<button class="btn">这个按钮只有在H5平台和App才能看到</button>
<!--  #endif -->

<!--  #ifdef  MP-WEIXIN -->
<view class="box">这个view只有在微信小程序平台才能看到</view>
<!--  #endif -->
