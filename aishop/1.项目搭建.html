# 2、运行项目

需求： 基于HbuilderX运行项目到微信开发者工具

思路：

  • 1、对微信小程序进行配置
  • 2、配置微信开发者工具路径和端口号
  • 3、点击运行项目

实现：

1、对微信小程序进行配置

  • 在manifest.json的微信小程序配置中，填入微信小程序ID

2、配置微信开发者工具路径和端口号

  • 在Hbuilder菜单工具-->设置-->运行配置 中配置微信开发者工具路径

  • 在微信开发者工具菜单设置-->安全设置中开启服务器端口

3、点击运行项目

  • HbuilderX中菜单运行-->运行到小程序和模拟器-->微信开发者工具

  • 注意： 如果启动失败，手动使用微信开发者工具导入unpackage/dist/dev下面的mp-weixin微信目录即可，因为这个就是打包微信小程序代码。

# 2、配置分包

需求： 为了防止微信小程序体积过大，超过2M无法发布，从而分包。

思路：

  • 1、设计分包

  • 2、配置分包

实现：

1、设计分包

  • 将项目设计3个分包，good分包，涉及商品相关，user分包，设计用户相关，order分包设计订单支付购买相关
  • 根目录创建pkgGood和pkgUser和pkgOrder这3个目录，作为分包目录

2、配置分包

  • 在page.json中，配置subPackages字段，配置一下内容

    "subPackages": [
    {
        "pages": [],
        "root": "pkgGood"
    },
    {
        "pages": [],
        "root": "pkgUser"
    },
    {
        "pages": [],
        "root": "pkgOrder"
    }
]

2、配置uview请求库实例

  • 创建utils/request.js文件，书写以下内容， 参考文档 (opens new window)

    // 为默认全局配置
uni.$u.http.setConfig((config) => {
	// config为默认全局配置
	config.baseURL = 'http://124.223.14.236:8361/api'; /* 根域名 */
	return config
})
	
// 请求拦截
uni.$u.http.interceptors.request.use((config) => {
	config.data = config.data || {}
	// 是否登录需要携带token
	// if(config?.custom?.auth) {
	// 	config.header.token = 'xxx'
	// }
	return config   // 一定要return config
}, config => { 
	return Promise.reject(config)
})
	
// 响应拦截
uni.$u.http.interceptors.response.use((response) => {
	const data = response.data  // 解构出来数据
	return data
}, (response) => { 
	return Promise.reject(response)
})
