# 2、渲染页面数据

需求： 从商品详情页面，点击立即购买进入确认订单界面【addType是1】，从购物车界面点击立即计算进入确认订单界面【addTyp是0】。 渲染出对应的商品确认订单信息。

思路：

  • 分析接口，发现接口需要提交
    • addressId即地址ID,默认没有就是0
    • addType即添加类型，要么是1要么是0
    • type即是否团购，要么1是要么0否，这里写死0
  • data定义好相关变量
  • onLoad里面将addType记录下来
  • methods定义好获取确认订单信息方法
  • onShow生命周期里面调用获取确认订单信息方法
  • 渲染数据到页面模板

实现：

data定义好相关变量

data(){
    return {
        checkedGoodsList: [], // 订单中的商品
        checkedAddress: {}, // 选中收货地址
        goodsTotalPrice: 0.00, //商品总价
        freightPrice: 0.00, //快递费
        orderTotalPrice: 0.00, //订单总价
        actualPrice: 0.00, //实际需要支付的总价
        addressId: 0, // 地址ID
        goodsCount: 0, // 商品总数
        postscript: '', // 订单留言

        payMethodItems: [{

        {
            name: 1,
            value: '在线支付',
            checked: 'true'
         },
        ],
        payMethod: 1, // 默认支付方式
    }
}

methods定义好获取确认订单信息方法

methods:{
    // 获取确认订单信息
    async getCheckoutInfo() {
        let res = await uni.$http.get('cart/checkout', {
            params: {
                addType: this.addType,
                addressId: this.addressId,
                type: 0  // 是否为团购
            }
        })
        this.checkedGoodsList = res.checkedGoodsList
        this.checkedAddress = res.checkedAddress || {}   // 如果返回是0就表示获取确认订单信息的时候就没有提交地址ID 且该用户没有默认收货地址。
        this.actualPrice = res.actualPrice
        this.addressId=  res.checkedAddress === 0 ? 0 : res.checkedAddress.id
        this.freightPrice = res.freightPrice
        this.goodsTotalPrice = res.goodsTotalPrice
        this.orderTotalPrice = res.orderTotalPrice
        this.goodsCount = res.goodsCount
        this.outStock = res.outStock
        if (res.outStock === 1) {
            uni.showToast({
                icon:'error',
                title:'有部分商品缺货或已下架'
            })
        } else if (res.numberChange === 1) {
            uni.showToast({
                icon:'error',
                title:'部分商品库存有变动'
            })
        }
    },
}

async getCheckoutInfo() {

需求： 线下支付的方式下，点击提交订单按钮完成订单提交，跳转到结果页面。

思路： 所谓的线下支付 只需要发生一个提交订单的请求即可，后端去自动处理线下支付的流程。

实现：

  • 提交订单 线下方式的事件函数offlineOrder里面，完成提交订单操作 【/pkgOrder/checkout/checkout.vue】

    methods:{
	// 线下支付提交订单
    async offlineOrder(){
        console.log("线下支付提交订单");
        if (this.addressId <= 0) {
            return uni.showToast({
                title:'请选择收货地址',
                icon:'error'
            })
        }
        try{
            // 发生确认订单请求
            await uni.$http.post('/order/submit',{
                addressId: this.addressId,
                postscript: this.postscript,
                freightPrice: this.freightPrice,
                actualPrice: this.actualPrice,
                offlinePay: 1
            })
            // 成功提交订单了。抹掉了地址ID，已经没有意义。
            uni.setStorageSync('addressId', 0);
            uni.redirectTo({
                url: '/pkgOrder/payOfflineResult/payOfflineResult?status=1',
            })
        }catch(e){
            // 订单提交失败
            setTimeout(()=>{
                uni.redirectTo({
                    url: '/pkgOrder/payOfflineResult/payOfflineResult?status=0',
                })
            },300)
        }
    },
}

# 7.2、线上订单提交

需求： 线上支付的方式下，点击提交订单按钮完成订单提交，跳转到支付界面

思路： 只需要发生一个提交订单的请求即可，我们会得到一个订单信息。携带订单id、编号、价格进入到支付界面。

实现：

  • 提交订单 线下方式的事件函数submitOrder里面，完成提交订单操作，跳转到支付界面，携带订单id、编号、价格 【/pkgOrder/checkout/checkout.vue】

    methods:{
    // 线上支付提交订单
    async submitOrder(){
        console.log("线上支付提交订单");
        if (this.addressId <= 0) {
            return uni.showToast({
                title:'请选择收货地址',
                icon:'error'
            })
        }
        // 发生确认订单请求
        let res = await uni.$http.post('/order/submit',{
            addressId: this.addressId,
            postscript: this.postscript,
            freightPrice: this.freightPrice,
            actualPrice: this.actualPrice,
            offlinePay: 0
        })
        // 确认订单提交成功了,需要清除本地的勾选地址信息
        uni.removeStorageSync('addressId')
        // 跳转到线上支付界面
        uni.showToast({
            title:'订单提交成功',
            icon:'success',
            duration:500
        })
        setTimeout(()=>{
            const {order_sn,id,order_price} = res.orderInfo
            uni.redirectTo({
                url:`/pkgOrder/payOnline/payOnline?order_sn=${order_sn}&order_id=${id}&pay_price=${order_price}`
            })
        },500)
    }
}

# 7.3、线上订单支付

需求： 点击微信支付实现唤醒微信支付功能。

思路：

  • 将微信支付逻辑抽离到混入mixins/index.js 中，方便后期订单详情未支付，需要立即支付的使用。

  • 微信支付准备工作 【实际工作不需要我们操心，有人完成】

    • 企业主体微信小程序，并认证通过。

    • 企业申请了微信支付功能，并且得到了商户号和秘钥。 具体点击 (opens new window)。

    • 将相关信息配置在后台中

  • 发起支付预请求，提交订单id 给后端。后端返回支付api调用需要的加密参数

  • 调用uni.requestPayment()方法，实现微信支付唤醒。根据支付结果进入到支付结果页面进行展示。

实现：

将微信支付逻辑抽离到混入mixins/index.js 中

发起支付预请求，提交订单id 给后端。后端返回支付api调用需要的加密参数

调用uni.requestPayment()方法，实现微信支付唤醒。根据支付结果进入到支付结果页面进行展示。

export default {
	methods:{
		// 跳转登录的方法
		goUrl(url,needLogin=false,isSwitch=false){
			//.....
		},
		// 微信支付的方法
		async weixinPay(orderId) {
			// 微信支付的前提： 小程序ID主体为企业、且开通认证、且开通微信商户号功能
			// 1. 提交订单的ID给后端，后端返回支付所需要的加密参数
			let { pay_data } = await uni.$http.get('/pay/preWeixinPay',{ params:{ orderId } })
			// 2. 调用支付API， uni.requestPayment()
			uni.requestPayment({
				provider:'wxpay',
				timeStamp:pay_data.timeStamp,
				nonceStr:pay_data.nonceStr,
				package:pay_data.package,
				signType:pay_data.signType,
				paySign:pay_data.paySign,
				success:res=>{
					// 3. 【有时候】再次发生请求给后端确认订单是否支付成，后端返回支付成功
					// 支付成功，跳转支付成功界面
					uni.redirectTo({
						url: '/pkgOrder/payOnlineResult/payOnlineResult?status=1&order_id='+orderId,
					})
				},
				fail:()=>{
					// 支付失败，跳转支付失败结果
					uni.redirectTo({
						url: '/pkgOrder/payOnlineResult/payOnlineResult?status=0&order_id='+orderId,
					})
				}
			})
		}
	}
}
