    # 1、小程序概述

    问题：

    1、什么是微信小程序

    2、与普通网页开发区别

    3、微信小程序开发流程

    总结：

    1、什么是微信小程序

    小程序是一种不需要下载、安装即可使用的应用，它实现了应用触手可及的梦想，用户扫一扫或者搜一下就能打开应用，也 实现了用完即走的理念，用户不用安装太多应用，应用随处可用，但又无须安装卸载 ---------微信之父、Foxmail 之父 – 张小龙

    特点：体积小、方便获取和传播；用完即走

    2、与普通网页开发区别

    • 运行环境不同： 网页运行在浏览器环境中, 小程序运行在微信环境中
    • 提供的API不同： 小程序中，无法调用 DOM 和 BOM 的 API， 却可以调用微信环境提供的各种组件、API，如：微信支付扫码等
    • 开发模式不同： 网页的开发模式：浏览器 + 编辑器 小程序拥有自己的一套标准开发模式：账号申请 + 开发工具 + 创建配置项目

    3、微信小程序开发流程

    • ①、注册小程序 在微信公众平台注册小程序，完成注册后可以同步进行信息完善和开发。
    • ②、小程序信息完善 填写小程序基本信息，包括名称、头像、介绍及服务范围等。
    • ③、开发小程序 绑定小程序开发者、开发信息配置、下载开发者工具、参考开发文档进行小程序的开发和调试
    • ④、提交审核和发布 完成小程序开发后，提交代码至微信团队审核，审核通过后即可发布（公测期间不能发布）

    总结概况

    # 4、宿主环境介绍

    问题：

    1、宿主环境是什么

    2、小程序的宿主环境是什么

    3、小程序的运行机制是什么

    总结：

    1、宿主环境是什么

    宿主环境指的是程序运行所必须的 依赖环境

    例如：Android 系统和 IOS 系统是两个不同的宿主环境，安卓版的微信 App 是不能在 IOS 环境下运行的。所以 Android 是安卓软件的宿主环境，脱离了宿主环境的软件是没有任何意义的

    2、小程序的宿主环境是什么

    微信是小程序的宿主环境，脱离了微信，小程序是运行不成功的

    小程序借助宿主环境提供的能力，可以完成许多普通网页无法完成的功能，例如：微信扫码、微信支付、微信登 录、地理定位 等等功能

    3、小程序的运行机制是什么

    1. 小程序启动的过程： ①. 把小程序的代码包下载到本地 ②. 解析 app.json 全局配置文件 ③. 执行 app.js 小程序入口文件，调用 App() 创建小程序实例 ④. 渲染小程序首页 ⑤. 小程序启动完成

    2. 页面渲染的过程：

      ①. 加载解析页面的.json 配置文件 ②. 加载页面的 .wxml 模板和 .wxss 样式 ③. 执行页面的 .js 文件，调用 Page() 创建页面实例 ④. 页面渲染完成

    思考

    ❓小程序的宿主环境是什么 ？

    点击查看答案

    小程序的宿主环境是微信

      ① 运行开发者工具，使用手机微信进行扫码，同时点击确认登录

    ​ ② 登录以后，会进入新的界面，此时便可以创建、编辑、调试小程序项目

    开发者工具的 5 个组成部分

    • 调试项目
    1. 编辑代码后保存或点击编译，即可在开发者工具的模拟器中查看到编辑后的效果
    2. 编辑代码保存后点击预览，使用微信扫一扫弹出的二维码，即可预览最新的效果

    2、有哪些核心文件和配置

    • 目录构成

    • 页面构成

    小程序官方建议把所有的小程序页面，都存放在 pages 目录中，每个页面以单独的文件夹存在

    每个页面由 4 个基本文件组成，分别是:

    .js 文件 - 页面的JS脚本文件

    .json 文件 - 页面的配置文件

    .wxml 文件 - 页面的模板结构

    .wxss 文件 - 当前页面的样式

    3、如何创建页面

    • 方式1：创建文件夹方式

      在 pages 文件夹上，右键新建文件夹，创建 home文件夹 ，在 home 文件夹右键新建 page 选项，输入 home

      • 官方推荐文件夹名称和页面的名称一致，方便后期的维护
      • 新建页面组件的时候，不需要输入后缀名

    • 方式2：配置 pages 字段方式

      • 在 app.json --> pages 中新增页面的存放路径，小程序开发者工具即可帮我们自动创建对应的页面文件
      • 对 pages 目录下页面的删除或者新增都需要在 app.json 文件的 pages 数组中进行新增或删除

    ​ **注意：**如果删除了一个页面，一定要去app.json中pages数组里面手动删除该页面配置项）

    4、如何配置首页

    • 方式1：调整 pages 字段

      调整 app.json → pages 数组中页面路径的前后顺序，即可修改项目的首页，小程序会把排在数组第一位的页面，当作项目首页进行渲染

    • 方式2：增加entryPagePath 节点

      在 app.json 中 pages 同级，配置一个新的节点 entryPagePath 节点，传入需要设置为首页的路径即可

    注意：最终小程序项目的首页，以 entryPagePath 节点设置的为准

    思考

    ❓ 小程序的 设置首页的两个方式是什么 ？

    点击查看答案
    1. 将首页的路由放到 app.json → pages 数组第一 项
    2. 配置一个新的节点 entryPagePath 节点

    # 2、组件介绍

    问题：

    1、什是wxml

    2、wxml和html区别

    3、有哪些核心的组件

    总结：

    1、什是wxml

    WXML（WeiXin Markup Language）是框架设计的一套标签语言（组件），用来构建小程序页面的结构， 其作用类似于网页开发中的 HTML

    文档地址： https://developers.weixin.qq.com/miniprogram/dev/component/

    2、wxml和html区别

    • 标签名称不同

    • 属性节点不同

    • 提供了类似于 Vue 中的模板语法

      • 数据绑定
      • 列表渲染
      • 条件渲染

    3、有哪些核心的组件

    • 常用的视图容器类组件

      ①、view 组件，类似div

      ②、scroll-view 组件，滚动容器，常用来实现滚动列表效果

      ③、 swiper 和 swiper-item 组 件，轮播组件，用来做轮播

    • 常用的基础内容组件

      ①、text 组件：类似span

      ②、rich-text 组件：类似v-html作用

    • 导航组件

      navigator组件：类似a标签

    • 常用表单组件

      ①、button组件：按钮组件

      ②、input 组件：输入框组件

      ③、checkbox 组件：多选组件

    • 媒体组件：

      ①、image组件：类似img

      ②、video组件：视频播放组件

      ③、camera组件：相机组件

    # 3、样式介绍

    问题：

    1、什么是wxss

    2、wxss和css区别

    3、rpx单位如何使用

    总结：

    1、什么是wxss

    WXSS (WeiXin Style Sheets)是一套样式语言，用于描述 WXML 的组件样式，类似于网页开发中的 CSS

    2、wxss和css区别

    • ①、新增了 rpx 自适应尺寸单位
      • CSS 中需要手动进行像素单位换算，例如 rem
      • WXSS 在底层支持新的尺寸单位 rpx，在不同大小的屏幕上小程序会自动进行换算
    • ②、提供了全局的样式和局部样式
      • 项目根目录中的 app.wxss 会作用于所有小程序页面
      • 局部页面 .wxss样式仅对当前页面生效
    • ③、 WXSS 仅支持部分 CSS 选择器
      • class 和 #id
      • element
      • 并集选择器和后代选择器
      • ::after 和 ::before 等伪类选择器
    • ④、原生微信小程序里不能使用相对图片作为背景图，要么使用网络图片，要么使用base64资源。目的是减少体积。

    3、rpx单位如何使用

    如果设计稿是iPhone6为标准的(真实宽度375，设计图宽度750)，则设计图里多少px代码就写多少rpx，屏幕变了，尺寸自动调整。

    # 4、Api介绍

    问题：

    1、微信小程序Api的作用

    2、微信小程序Api的分类

    总结：

    1、微信小程序Api的作用

    通过这些丰富的微信原生 API，可以方便的调起微信提供的能力，如获取用 户信息，本地存储，支付功能等

    文档地址：https://developers.weixin.qq.com/miniprogram/dev/api/

    2、微信小程序Api的分类

    ①、事件监听 API

    • 特点：以 on 开头的 API 用来监听某个事件是否触发
    • 举例：wx.onWindowResize(function callback)

    ②、同步 API

    • 特点1：以 Sync 结尾的 API 都是同步 API
    • 特点2：同步 API 的执行结果，可以通过函数返回值直接获取，如果执行出错会抛出异常
    • 举 例：wx.setStorageSync('key', value)向本地存储中写入内容

    ③、异步 API

    • 特点：类似于 jQuery 中的 $.ajax(options) 函数，需要通过 success、fail、complete 接收调用结果
    • 举例：wx.request()发起网络数据请求，通过 success 回调函数接收数据
    • 小程序中，大多数的 API 都是异步 API

    • ① 进入小程序所在目录，执行npm init -y实现将项目初始化为包环境

    • ③ 构建 npm 包 ，使用 npm 模块 。然后就会出现 一个miniprogram_npm目录，里面存放构建结果

      注意：

      • node_modules 不会被打包上传，只是从中构建出内容放在miniprogram_npm目录中
      • 建议每次安装包之前先把 ``miniprogram_npm`删除，然后重新构建，保证构建有效性

    • 开发者：在小程序早期负责开发的人员 【我们前端的角色】
