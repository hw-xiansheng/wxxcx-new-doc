    # 一、页面导航

    思考

    ❓ Vue 中实现导航跳转的方式有几种，分别是什么？

    点击查看答案

    Vue 中实现导航跳转的方式有 2 种

    • 声明式导航 <router-link to="/地址"></router-link>

    • 编程式导航 router.push('/地址')

    小程序里面同样也有2种方式

    # 1、声明式导航

    **目标：**如何实现声明式导航、如何实现页面传参

    实现：

    （1）、如何实现声明式导航

    通过点击 <navigator> 组件实现页面跳转，具体参见文档 【组件=>导航=>navigator】 (opens new window)

    <navigator url="/页面地址" open-type="打开类型">跳转页面</navigator>
    1

    注意：

    • url 表示要跳转的 页面的地址，必须以 /开头
    • open-type 表示 跳转的方式
      • switchTab跳转到tabBar页面
      • navigate跳转到常规页面，默认值
      • 其他查阅文档

    （2）如何实现页面传参

    url上面查询参数绑定数据

    <navigator url="/页面地址?key=val&key2=val2">跳转页面</navigator>
    1

    对应页面的js文件里面读取

    Page({
	onLoad(options){
        // options就是页面传递过来的数据   {key:val,key2:val2}
    }
})

    注意： 页面传参无法给tabBar页面传递参数

    • 1、如何实现声明式导航

      url 必须以 /开头，open-type 跳转方式，switchTab跳转到tabBar页面，navigate跳转到常规页面

    # 二、页面事件

    # 1、下拉刷新

    思考

    ❓ 什么是下拉刷新

    点击查看答案

    下拉刷新 是移动端的专有名词，指的是通过手指在屏幕上的下拉滑动操作，从而 重新加载页面数据 的行为

    问题：

    1、如何开启下拉刷新

    2、如何配置窗口样式

    3、如何监听下拉刷新

    4、如何停止下拉刷新

    总结：

    1、如何开启下拉刷新

    三种方式，分别是：

    • 全局开启下拉刷新

      在app.json的 window节点中，将 enablePullDownRefresh设置为 true

    • 局部开启下拉刷新

      在页面的.json 配置文件中，将 enablePullDownRefresh设置为true

    • 使用小程序提供的 API 实现下拉刷新

      wx.startPullDownRefresh()  // 主动触发下拉刷新，就像手指下拉一样的效果
      1

      注意： pc模拟器里面是无效的，手机真机是可以实现该效果

    2、如何配置窗口样式

    在全局或页面的.json 配置文件中配置以下几个参数



     
     


    {
  "enablePullDownRefresh": true,  // 页面开启下拉刷新
  "backgroundColor":"#ff0000",  // 背景色
  "backgroundTextStyle":'light' // loading 样式 只有 light和dark
}

		count:0

    注意：下拉刷新的loading 效果会一直显示，不会主动消失， 所以需要手动隐藏loading效果

    调用 wx.stopPullDownRefresh()可以停止当前页面的上拉刷新

    # 三、生命周期

    思考

    ❓ Vue 中生命周期几个阶段；几个钩子函数

    点击查看答案

    Vue 中实现导航跳转的方式有 2 种

    • 四个阶段（创建阶段、挂载阶段、更新阶段、销毁阶段）

    • 八个钩子（beforeCreated、created、beforeMount、mounted、beforeUpdate、update、beforeDestory、destoryed）

    小程序里面同样也有生命周期，并且分为2类。

    # 1、应用生命周期

    问题：

    1、含义是什么？

    2、有哪些函数？

    总结：

    1、含义

    • 特指小程序从启动 --> 运行 --> 销毁期间依次调用的那些函数

    2、钩子函数

    // 写在App.js中
App({
  /**
   * 当小程序初始化完成时，会触发 onLaunch（全局只触发一次）

  },
  /**
   * 当小程序启动，或从后台进入前台显示，会触发 onShow

     console.log("【应用生命周期】===onShow")

   */
  onHide: function () {
      // todo：每次离开小程序都做的事情
     console.log("【应用生命周期】===onHide")
  },
  /**
   * 当小程序发生脚本错误，或者 api 调用失败时，会触发 onError 并带上错误信息

    # 2、页面生命周期

    问题：

    1、含义是什么？

    2、有哪些函数？

    总结：

    1、含义

    • 特指小程序中，每个页面从 加载 --> 渲染 --> 销毁期间依次调用的那些函数

    2、钩子函数

    Page({
    // 生命周期函数--监听页面加载，一个页面只调用1次
    onLoad:function(options){
        // options接收传递进入页面的数据
        console.log("【页面生命周期】===onLoad")
    },
    // 生命周期函数--监听页面显示
    onShow:function(){
        console.log("【页面生命周期】===onShow")
    },
    // 生命周期函数--监听页面初次渲染完成，一个页面只调用1次
    onReady:function(){
        console.log("【页面生命周期】===onReady")
    },
    // 生命周期函数--监听页面隐藏
    onHide:function(){
        console.log("【页面生命周期】===onHide")
    },
    // 生命周期函数--监听页面卸载
    onUnLoad:function(){
        console.log("【页面生命周期】===onUnLoad")
    }

    # 3、生命周期

    小程序组件生命周期分为2类： 一类是组件生命周期，一类是组件所在页面生命周期

    组件核生命周期有created,attached,ready,moved,detached,error

        },
        detached(){
            // todo: 一些组件销毁后要做的事情
            console.log('在组件实例被从页面节点树移除')
        }
    }
})

Component({
  // ... 其他

   * 组件的方法列表

  methods: {
    changeTab(event){
      // 取出索引和跳转地址
      const { index, url } = event.currentTarget.dataset
      // 修改激活索引
      this.setData({ activeIndex:index })
      // 跳转界面
      wx.switchTab({  url:'/'+url })  // 注意跳转地址前面要 /
    }
  }
